Protocol

Demo21 & 22 & 23

  • Protocol: 协议
  • 略类似 Java 接口
  • 可以声明一些必须实现的方法和选择实现的方法

  • 作用：

    • 用来声明一些方法（即 Protocol 是由一系列方法声明组成）

  • 格式

  • 注意点

    • 继承默认声明＋实现，协议只有声明
    • 相同类型的类可以继承，不同类型只能使用 Protocol
    • Protocol 可以用于存储方法的声明，可以将多个类中共同方法抽取出来，以后让这些类遵守协议即可
    • 协议只能声明方法，不能声明变量，不能写实现
    • 父类遵守了某个协议，那么子类也会自动遵守这个协议
    • 在 Obj-C 中一个类可以遵守一个或多个协议（Obj-C 中的类只能有一个父类，即只有单继承）
    • Obj-C 中的协议又可以遵守其它协议，只要一个协议遵守了其它协议，那么这个协议中就会自动包含其它协议的声明

  • 基协议

    • NSObject: 可省略

  • @required & @optional 关键字

    • @required: 默认，必须实现
    • @optional: 不一定实现
    • 但如果不实现也只是警告，并不是报错，因此该关键字不能严格控制

  • 应用场景

    • 类型限定
    • 可以将协议写在数据类型的右边，明确的标注如果想给该变量赋值，那么该对象必须遵守某个协议，若不遵守将会警告

  • 注意点

    • 虽然在接受某一个对象的时候，对这个对象进行了类型限定（限定它必须实现某个协议），但是并不意味着这个对象就真正的实现了该方法，所以每次在调用对象的协议方法时应该进行一次验证

  • 应用场景

    • 代理设计模式
    • 场合（让对象 B 成为对象 A 的代理对象）
    • 当对象 A 发生了一些行为，想告诉对象 B
    • 对象 B 想监听对象 A 的一些行为
    • 当对象 A 无法处理某些行为时，想让对象 B 帮忙处理

  • 协议的编写规范

    • 一般情况下，当前协议属于谁，就将协议定义到谁的头文件中
    • 协议的名称一般以所属类的类名开头，后面跟上 protocol 或者 delegate
    • 协议中的方法名称一般以所属类的类名作为开头
    • 一般情况下协议中的方法会将触发该协议的对象传递出去
    • 一般情况下一个类中的代理属于的名称叫做 delegate
    • 当某一个类要成为另外一个类的代理的时候，一般情况下在 .h 中用 @protocol 协议名称; 告诉当前类这是一个协议。在 .m 中用 #import 真正的导入一个协议的声明

  • 字符串存储位置

    • 字符串常量－常量区
    • alloc initWithFormat / stringWithFormat －堆
    • 注意:
    • 不同的平台存储的方式不一样：
    • 此处有疑问
    • 不同的编译器存储的方式不一样：
    • 如果是 Xcode 6 以下并且是在 iOS 平台，那么每次 alloc 都会创建一个新的对象，如果是在 Xcode 6 以上那么 alloc 多次指向同一块存储空间

  • 如果是通过字符串常量创建两个对象，并且字符串常量的内容一致，那么如果创建多个字符串对象，多个对象指向同一块存储空间

  • 只要调用 alloc 就会在堆内存中开辟一块存储空间，如果是 Mac 平台系统会自动对字符串对象进行优化，但是如果是 iOS 平台就是两个对象

  • 注意

    • 一般情况下，只要是通过 alloc 或者类工厂方法创建的对象，每次都会在堆内存中开辟一块新的存储空间，但是如果是通过 alloc 的 initWithString 方法除外，因为这个方法是通过 copy 返回一个字符串对象，而 copy 又分为深拷贝和浅拷贝，如果是深拷贝会创建一个新的对象，如果是浅拷贝不会创建一个新的对象，而是直接返回被拷贝的对象的地址

有点凌乱

  • URL
    • 基本 URL 包含协议，主机域名（服务器名称或 IP 地址），路径

  • URL 协议头

    • http:// https:// 超文本传输协议资源
    • ftp:// 文件传输协议
    • file:// 本地电脑的文件

  • 注意

  • 如果通过 fileURLWithPath: 方法创建 URL，系统会自动添加协议头（file://）
  • 开发中一般情况下，如果是访问本机的资源，创建 URL 时候，建议使用 fileURLWithPath 方法创建，因为 URL 不支持中文，但是如果通过 fileURLWithString 方法创建，可以包含中文，系统内部会自动对其进行处理
  • 如果多次往同一个文件中写入内容，那么后一次的会覆盖前一次的
