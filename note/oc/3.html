封装 & getter-setter

Demo02

  • 封装

    • 隐藏对象的属性和实现的细节，仅对外提供公共的访问方法
    • 好处：提高安全性；将变化隔离
    • 类是对数据与功能的封装，数据即成员变量，功能即类／对象方法
    • 成员变量的封装是对数据的封装

  • 成员变量以下划线开头的好处：可以区分局部变量和成员变量

  • 遵循驼峰命名法

  • setter 方法

    • 作用：设置成员变量的值
    • 格式：
    • 对象方法
    • 没有返回值
    • 方法名称以 set 开头，并加上去掉下划线的成员变量名

  • getter 方法

    • 作用：获取成员变量的值
    • 格式：
    • 对象方法
    • 没有参数，有返回值，注意类型一致
    • 方法名称为去掉下划线的成员变量名

  • 若某一属性只提供了 getter 方法，称之为只读属性

  • 若某一属性只提供了 setter 方法，称之为只写属性
  • 若某一属性同时提供 setter & getter 方法，称之为可读可写属性

  • 若某一属性 getter & setter 均没有，称之为私有属性

  • 点语法

  • 点语法的本质是调用了 setter/getter 方法

  • 点语法是一个编译器的特性，会在程序翻译成二进制的时候将点语法自动转换为 setter 和 getter 方法（将其转为 [对象名 方法名]）

    • 如果点语法在 = 号的左边，那么编译器会自动转换为 setter 方法
    • 如果点语法在 = 号的右边，或者没有等号，那么编译器就会自动转换为 getter 方法

  • 点语法注意点：一般用于给成员变量设值，如果不是给成员变量赋值一般情况下不建议使用，但是也可以使用（例如代码中的 test 方法）

self

Demo03

  • 若 self 在对象方法中，那就代表调用当前对象方法的对象

  • 若 self 在类方法中，那就代表调用当前类方法的类

  • 使用场景

    • 可以用于在对象方法之间进行相互调用
    • 可以用于在类方法之间进行相互调用
    • 可以用于区分成员变量和局部变量同名的情况（实际使用仍有警告，所以不建议）

  • 注意点

    • self 会自动区分类方法和对象方法，如果在类方法中使用 self 调用对象方法，那么会直接报错
    • 不能在对象方法或者类方法中利用 self 调用当前所在的方法（死循环）

继承 & super

Demo04

  • 继承语法

    • @interface 子类名 : 父类名
    • @implementation 子类名 : 超类名

  • 继承自 NSObject 的类

    • new 方法为继承来的
    • isa 指针也是继承的

  • 子类拥有父类的所有属性和方法（对象方法和类方法）

  • 如果子类中有和父类中同名的方法，那么我们称之为方法重写（成员变量不可重写，因为继承的成员变量是拷贝的形式）

  • 继承中的方法调用顺序

    • 如果自己有就调用自己的，如果自己没有就调用父类的（若没有再父类的父类…）如果一直找到 NSObject 类都没有找到，那么就会报错
    • 报错：unrecognized selector sent to instance ...

  • 继承

    • 优点：提高代码的复用性
    • 缺点：耦合性太强（依赖性太强）
    • 使用条件：
    • XXX 是 XXX / 某某某 is a 某某某

  • 继承可以让类与类之间产生关系，正是因为继承让类与类之间产生了关系所以才有了多态

  • 可以利用 super 在任意方法中调用父类中的方法

    • super在类方法中，一定会调用父类的类方法
    • super在对象方法中，一定会调用父类的对象方法

多态

Demo05

  • 多态：事物的多种表现形态
    • 动态类型
    • 编译时编译器只会检查当前类型有无调用的方法，运行时，系统会自动判断真实类型
    • 父类指针指向子类对象，但如果要调用子类特有方法，需要强制类型转换

成员变量修饰符

Demo06

  • @public

    • 可以：其它类，本类，子类

  • @private

    • 不可以：其它类，子类
    • 可以：本类

  • @protected

    • 不可以：其它类
    • 可以：本类，子类
    • 注意：默认情况下所有的实例变量都是 @protected

  • @package（暂不了解）

    • 介于 @public 和 @private 之间
    • 在其它包中访问－@private
    • 在当前代码所在包中访问－@public

  • 作用域：

    • 从出现的位置开始，一直到下一个修饰符出现
    • 如果没有遇到下一个实例变量修饰符，那么就会修饰后面所有的实例变量

description

Demo07

  • description

    • %@ 用来输出对象
    • 用 %@ 打印对象，默认调用父类的 description 方法
    • 默认格式：<类名: 对象地址>

  • [类名 class] 返回该类的类对象

  • 打印实例对象，调用对象(-)方法

  • 打印类（对象），调用类(+)方法（不常用）

  • 注意点：

    • description 死循环
    • return [NSString stringWithFormat:@"%@", self];
