导入外部的类

  • #import & #include

    • #import: Obj-C 引入的，可导入 C/Obj-C 的头文件。可以判断导入的类是否重复
    • #include: 继承自 C 的，不能判断重复导入

  • "XXX.h" & <XXX.h>

    • "XXX.h": 当前目录－（编译器）类库
    • <XXX.h>: （编译器）类库

  • <Foundation/Foundation.h>

    • Foundation.h: 主头文件
    • 所有的主头文件的名称都和工具箱的名称一致
    • 所有的主头文件都是导入了该工具箱中所有工具的头文件
    • 工具箱的地址: /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS.sdk/System/Library/Frameworks

面向对象

  • 面向对象编程

    • Object Oriented Programming(OOP)

  • 面向过程：

    • 关注的是解决问题需要哪些步骤

  • 面向对象：

    • 关注的是解决问题需要哪些对象

  • 类与对象

    • 定义类：成员（变量＋方法）
    • 一般名词为类
    • 将拥有相同成员变量（属性）和成员方法（行为）的对象抽象为类

  • 注意点

    • 定义行为（成员方法）时，谁最了解此行为，则此行为属于哪个类

类

  • 声明
    • 声明类中的成员变量和成员方法

  • 实现

    • 实现类中的成员方法

  • 创建对象（new)

    • 分配存储空间
    • 初始化成员变量（默认 0）
    • 返回内存首地址（isa）

  • 规范

    • 类名各单词首字母大写
    • 类成员变量名以 _ 开头

  • Obj-C 的类与对象

    • 如果使用了一个指针保存了某一个对象的地址，那么这个指针称之为某类型的对象
    • Obj-C 类的本质是结构体

方法

  • 方法的声明

    • 命名使用驼峰命名法
    • 数据类型使用 () 括住
    • 每个形参的数据类型前加 :
    • Obj-C 允许在第二个及以后的参数数据类型前加标签
    • : 和标签也属于方法名

  • 类方法（+）

    • 只能由类调用
    • 不可直接访问成员（变量＋方法）
    • 效率高（所以方法中若没有使用到成员变量，尽量使用类方法）
    • 一般用于定义工具方法：字符串查找、文件操作、数据库操作

  • 对象方法（-）

    • 只能由对象调用
    • 可以直接访问当前对象的成员（变量＋方法）
    • 可与类方法同名

  • 注意点

    • 在 Obj-C 中，方法调用又称消息传递（Message Passing）
    • 类方法和对象方法可相互调用，但是最好不要在类方法里调用对象方法（需要再次创建对象）

内存

  • isa

    • isa 是一个指针，占 8 个字节
    • 创建对象的时候返回的地址是类的第 0 个成员变量 isa 的地址

  • 类／对象方法调用细节

    • 类方法（效率高）
    • 通过类名找类对象－类对象的方法列表
    • 对象方法
    • 通过指针找对象－对象通过 isa 指针找类对象－类对象方法列表

各种变量

  • 成员变量

    • 写在类声明中的变量
    • 不能在定义时初始化
    • 作用域：当前对象
    • 存储：当前对象对应的堆内存（需要手动释放）

  • 局部变量

    • 写在函数或者代码块的变量
    • 作用域：从定义的那行起到大括号或 return
    • 存储：栈内存（系统自动释放）

  • 全局变量

    • 写在函数和大括号外的变量
    • 作用域：从定义的那行起到文件末尾
    • 存储：静态区（程序一启动就会分配空间，直到程序结束释放）

函数与方法

  • 区别

    • 函数属于整个文件（甚至可以放在类的实现里，但放在声明中不能识别）
    • 方法属于某个类
    • 函数可以直接调用
    • 方法必须被对象或类调用

  • 注意点

    • 方法可以只有声明，没有实现（编译不会报错，但运行时调用了会出错）
    • 错误信息：
    • 对象方法：unrecognized selector sent to instance ...
    • 类方法：unrecognized selector sent to class ...
    • 方法也可以只有实现，没有声明
    • 类可以只有实现，没有声明（需要在实现时 : NSObject）
    • 在开发中不建议以上不规范的写法

常见错误

  • @interface @end 和 @implementation @end 不能嵌套使用
  • Obj-C 是弱语法，可以只有 @implementation，但开发中不建议
  • 漏写 @end
  • 成员变量没有写在 {} 中
  • 成员方法声明写在了 {} 中
  • 成员变量声明时不能对其初始化
  • 不能像函数一样调用方法
  • 成员变量和成员方法不能用 static/extern 等关键字修饰（不要和 C 语言混淆）
  • 类的实现可以写在 main 函数后面，在使用之前有声明即可
