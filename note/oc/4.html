@property

Demo09

  • Xcode 4.4 之前：

    • @porperty
    • 是一个编译器指令
    • 自动生成 getter/setter 方法声明
    • synthesize
    • 是一个编译器指令
    • 自动生成 getter/setter 方法实现

  • Xcode 4.4 之后：

    • @porperty
    • 自动生成 getter/setter 方法声明和实现
    • 只会生成最简单 getter/setter 方法，不能过滤传入的数据

  • 如果重写了 setter 方法，那就只生成 getter

  • 如果重写了 getter 方法，那就只生成 setter

  • 如果同时重写了 getter/setter 方法，那就不会自动生成私有的成员变量

  • @property(属性修饰符, 属性修饰符, ...) 数据类型 变量名;

  • readonly

    • 只生成 getter 方法

  • readwrite（默认）

    • 同时生成 getter/setter 方法

  • 指定生成的 getter/setter 方法名

    • 一般情况下获取布尔类型的属性的值，都会将获取的方法名称改为 isXxx
动态数据类型 id

Demo10

  • id 基本用处：

    • 定义变量
    • 作为函数的参数
    • 作为函数的返回值

  • 静态数据类型

    • 在编译时就知道变量的类型，其中属性和方法
    • 在编译时可以访问属性和方法
    • 若调用子类特有的方法，编译器就会报错

  • 动态数据类型 id

    • 在编译时编译器不知道变量的真实类型，只有在运行时才知道它的真实类型
    • 若调用子类特有的方法或私有方法，编译器不会报错
    • 注意：
    • id 中已经带有 *，所以只能指向 Obj-C 的对象
    • id 类型不能使用点语法，因为点语法是编译时特性，id 是运行时特性
    • 不要滥用动态数据类型，若没有用到多态，则尽量使用静态数据类型
    • 弊端：
    • 由于动态数据类型可以调用任意方法，所以有可能调用到不属于自己的方法，而编译时又不会报错，所以可能导致运行时的错误（为了尽可能减少编程中的错误，id 类型调用本项目所有类中都没有的方法时，编译器会报错）
    • 应用场景：
    • 多态：可以减少代码量，避免调用子类特有的方法时需要强制类型转换

  • id == NSObject * 万能指针

    • 区别：
    • NSObject * 是一个静态数据类型，不能调用子类特有的方法
    • id 是一个动态数据类型，能调用子类特有的方法

  • 为了避免动态数据类型引发的运行时的错误，一般情况下如果使用动态数据类型定义一个变量，在调用这个变量的方法之前会进行一次判断，判断当前变量是否能够调用这个方法

  • instancetype == id 万能指针

    • id 在编译时不能判断对象的真实类型
    • instancetype 在编译时可以判断对象的真实类型
    • id 可以用来定义变量，作为返回值，形参
    • instancetype 只能用于作为返回值

  • 自定义构造方法的返回值尽量使用 instancetype，不要使用 id

new & alloc & init

Demo11 & 12

  • new

    • 开辟内存空间
    • 初始化成员变量
    • 返回新对象的首地址
    • 相当于 alloc & init 结合体

  • alloc 必须还要调用 init，虽然 init 默认将当前对象返回

  • alloc 和 init 返回同一个地址

  • 构造方法 init

    • 用途：
    • 重写 init 用于初始化对象

  • 自定义构造方法

    • 对象方法
    • 返回 id/instancetype
    • 方法名以 init 开头
    • 必须遵循驼峰命名法

  • 一个类可以有 0 个或者多个自定义构造方法

  • 自定义构造方法可以有 1 个或多个参数

自定义类工厂方法

Demo13

  • 工厂方法（快速创建方法）
    • 用于分配，初始化实例并返回一个它自己的实例的类方法（eg. new）
    • 规范：
    • 类方法（+）
    • 返回值一般为 instancetype
    • 方法名以类名开头，遵循驼峰命名法

类的本质

Demo14

  • 类的本质

    • 类的本质是一个对象（类对象）
    • 程序中第一次使用该类时被创建，并在程序运行时一直存在，整个程序只有一份
    • 类对象是一种数据结构，存储类的基本信息：类的大小，类名称，类的版本，继承层次，消息与函数的映射表等
    • 类对象代表 Class 类型，对象方法属于类对象
    • 所有类的实例都由类对象声称，类对象会把实例的 isa 值改为自己的地址，使得每个实例指向该类的类对象

  • 类的内存细节

    • 暂略（补图）

  • 类的启动过程

  • load
    • 只要程序启动就会将所有类的代码加载到内存中，放到代码区
    • load 方法会在当前类被加载到内存的时候调用，有且仅调用一次
    • 如果存在继承关系，会先调用父类，再调用子类
  • initialize
    • 当类第一次被使用的时候就会调用（创建类对象的时候）
    • initialize 方法在整个程序的运行过程中有且仅调用一次
    • initialize 用于对某一个类进行一次性的初始化
    • 如果存在继承关系，会先调用父类，再调用子类

SEL 数据类型

  • SEL

    • SEL 类型代表方法的签名，在类对象的方法列表中存储着该签名与方法代码的对应关系
    • 每个类的方法列表都存储在类对象中
    • 每个方法都有一个与之对应的 SEL 类型的对象
    • 根据一个 SEL 对象就可以找到方法的地址，进而调用方法
    • SEL 类型定义：typedef struct objc_selector *SEL

  • [p test];

    • 首先把 test 这个方法名包装成 SEL 类型的数据
    • 根据 SEL 数据到该类的类对象上去找对应方法的代码
    • 若没有找到根据类对象上的父类的类对象指针，去父类的对象中查找，若没有就一直向上找，直到基类 NSObject
    • 若没有就报错，若有就执行
    • 注意（不需要担心性能问题）：
    • 这个操作过程中有缓存，第一次是一个一个找（耗性能），之后就可以直接调用

  • respondsToSelector

    • 如果通过对象来调用该方法会判断该对象有没有实现对象方法
    • 如果通过类来调用该方法会判断该类有没有实现类方法
