Objective-C Note

ARC

  • ARC 自动引用计数

    • ARC 是编译器特性，而不是运行时特性
    • 不同于其他语言的垃圾回收

  • ARC 判断原则

    • 只要还有一个强指针变量指向对象，对象就会保持在内存中

  • 强指针

    • 默认所有指针变量都是强指针
    • 被 __strong 修饰的指针

  • 弱指针

    • 被 __weak 修饰的指针

  • ARC

    • A 对象想用 B 对象，那么就需要用一个强指针指向 B 对象
    • A 对象不用 B 对象了，编译器会自动处理

  • ARC 下 @property 参数

    • strong 用于 Obj-C 对象，相当于 MRC 的 retain
    • weak 用于 Obj-C 对象，相当于 MRC 的 assign
    • assign 用于基本数据类型，同 MRC

category

  • 作用

    • 可以在不修改原来类的基础上，为类扩充方法
    • 将一个庞大的类分模块开发
    • 一个庞大的类可以由多人开发

  • 格式

    • 新添加的方法须写在 @interface @end 之间
    • ClassName：需要给哪个类扩充方法
    • CategoryName：分类的名称
    • newMethod：扩充的方法
  • 分类是用于给原有类添加方法的，它只能添加方法，不能添加属性(成员变量)
  • 分类中的 @property，只会生成 setter/getter 方法的声明，不会生成实现以及私有的成员变量
  • 如果分类中有和原有类同名的方法，会调用分类中的方法，也就是说会忽略原有类的方法（开发中不建议）

  • 如果多个分类中都有和原有类中同名的方法，那么调用该方法的时候执行谁由编译器决定，会执行最后一个参与编译的分类中的方法（开发中不建议）

  • 方法的调用顺序

    • 分类
    • 本类
    • 父类

  • 更改编译顺序

    • 总项目－当前项目－Build Phases－Compile Sources

  • Class Extension(匿名分类，延展) －Category 特例

    • 没有名字
    • 写在 .m 中
    • 可以为某个类扩充一些私有的成员变量及方法
    • 平时可以在 @implementation 中声明私有成员变量，只写实现作为私有方法

Block

Demo19 & 20

  • Block 是一种特殊的数据类型

  • 应用场景

    • 动画
    • 多线程
    • 集合遍历
    • 网络请求回调

  • 作用

    • 用来保存某一段代码，可以在恰当的时候取出来调用
    • 功能类似于函数和方法

  • 定义格式

  • 注意

    • 当发现代码的前面和后面都是一样的时候，这个时候就可以使用 Block
    • Block 中的变量和外界的变量并不是同一个变量
    • 如果 Block 中访问到了外界的变量，Block 会将外界的变量拷贝一份到堆内存中
    • 因为 Block 中使用的外界变量是拷贝的，所以在调用之前修改外界变量的值，不会影响到 Block 中拷贝的值
    • 在外界变量前面加上 __block 将可以修改外界变量的值
    • 如果在 block 中修改了外界变量的值，会影响到外界变量本身的值，因为加上 __block 之后是地址传递

  • cc -rewrite-objc "文件名.m": 可以将 Obj-C 反编译为 C++

  • MRC（仅做记录，不深究）

    • 默认情况下 block 存储在栈中，如果对 block 进行一个拷贝操作，block 会转移到堆中
    • 如果在栈中，block 中访问了外界的对象，那么不会对对象进行 retain
    • 如果在堆中，block 中访问了外界的对象，那么会对外界的对象进行一次 retain
    • 如果在 block 中访问了外界的对象，一定要给对象加上 __block，只要加上了 __block，哪怕 block 在堆中，也不会对外界的对象进行 retain
