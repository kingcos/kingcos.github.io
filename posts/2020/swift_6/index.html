Swift 拾遗 - 结构体 Struct & 类 Class

2020-02-09 — 4 min read
#Swift 

结构体

内存布局

struct Foo {
    var a = 0
    var b = 0
}

var foo = Foo()

print(MemoryLayout<Foo>.size)      // 16
print(MemoryLayout<Foo>.stride)    // 16
print(MemoryLayout<Foo>.alignment) // 8


func stack() {
    struct Foo {
        var a: Int
        var b: Int
    }

    let foo1 = Foo(a: 10, b: 11) // BREAKPOINT ⚠️
    var foo2 = foo1

    foo2.a = 20
    foo2.b = 22
}

stack()

我们将结构体定义在函数内部，并在函数体中声明并初始化结构体变量，此时 foo1 和 foo2 的内存空间均开辟在栈区。我们可以在汇编代码中清楚得看到这一过程：

我们尝试定义在全局范围中：

struct Foo {
    var a: Int
    var b: Int
}

let foo1 = Foo(a: 10, b: 11)
var foo2 = foo1

foo2.a = 20
foo2.b = 22

  • 规律
    • 变量的内存地址格式为：0x4bdc(%rip)，一般是全局变量，全局区（数据段），代码一旦装载，地址即确定，因此 rip 确定
    • 变量的内存地址格式为：-0x78(%rbp)，一般是局部变量，栈空间，rbp 则根据调用，每次都会不一样，因此局部变量的内存地址不确定
    • 变量的内存地址格式为：0x10(%rax)，一般是堆空间
