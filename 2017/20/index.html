Preface

CS193p 是斯坦福大学的一门公开课，今年 iOS 10 & Swift 3.0 的版本如约而至，还是 Paul 老爷爷带课。之前虽然也有听过他的课，但没有坚持下来，也没有做相应的笔记。为了方便交流分享，我在 GitHub 建立了一个 Repo：https://github.com/kingcos/CS193P_2017，会将课上的代码 Commit，也会分享笔记、心得。

Memory Management

内存管理也是 iOS 中不可回避的问题，但由于我个人能力有限，这里只记录了老师所讲的点，未来可能会再进行总结。

  • ARC: Automatic Reference Count 自动引用计数（!= Garbage Collection 垃圾回收）
  • 引用类型（例如类）存储在堆（Heap）中。

strong

  • 「强」引用：
    • 默认的引用指针，可省略
    • 只要有强引用指针指向，对象将一直保存在堆中。

weak

  • 「弱」引用：
    • 当对象没有被使用时，即被销毁（nil）。
    • 弱引用用于指向引用类型的可选类型指针。
    • 弱引用指针将不会把对象保存在堆中。
  • 例子：
    • outlets（其被视图层次强力持有，所以可为 weak）。

unowned

  • 「不」持有：
    • 需确保指针指向的对象没有被销毁（离开堆），否则程序会崩溃。
    • 常只用于打破循环引用。
